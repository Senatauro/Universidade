1)Explique o conceito de coesão e consistência no contexto de projeto de software. 
    Coesão e consistencia são termos relacionados a equipes de desenvolvimento, aonde uma equipe que contem o mesmo pensamento para o desenvolvimento do projeto, não se contradiz e está unida é uma equipe coesa e coerente

2)Explique o princípio da ocultação de informações, conforme se aplica ao projeto de software (relacionado com modularização). 
    A ocultação de informações é o processo de encapsular informações de uma classe para somente a mesma ter acesso livre a seus dados enquanto outras podem acessar tais informações por meio de funções auxiliares

3)Liste os quatro modelos de projeto necessários para uma especificação completa de um projeto de software e o papel de cada um.
    Dados:
        Criação de um modelo de dados e/ou informações que é representado em um nivel de abstração elevado
        No nivel de arquitetura, o projeto de dados se concentra em arquivos ou banco de dados
        No nivel de componentes, o projeto de dados considera as estruturas de dados necessarias para implementar os objetos de dados locais

    Arquitetura:
        Padrões de arquitetura definem uma abordagem especifica para tratar alguma caracteristica do sistema
        
    Componentes:
        Os padrões de projeto de componentes dão soluções comprovadas que tratam de um ou mais subproblemas extraidos do modelo de requisitos.
        Em muitos casos, os padrões de projeto desse tipo focam em algum componente funcional do sistema.

    Interface:
        Utilizado para representar fluxos de informações que entram e saem de um sistema e como são transmitidos entre os componentes definidos como parte da arquitetura

4)Quais são as diferenças importantes entre o gerenciamento de projetos de software e outros tipos de gerenciamento de projetos?
    O projeto de software não lida com um produto fisico, mas sim intangivel.
    Não existem padrões os quais podem ser reutilizados entre vários projetos.
    Cada projeto é unico em suas caracteristicas e requisitos.


5)Liste 4 atividades fundamentais de gerenciamento de projetos.
    Gerenciamento de riscos, recursos, planejamento de projeto e elaboração de propostas
